# GEMINI.md

## Project Overview
SuperCopy is a high-performance file utility for Windows, designed to accelerate file copying and provide archive unpacking capabilities. It features both a modern Graphical User Interface (GUI) and a Command-Line Interface (CLI).

- **Purpose:** Accelerated multi-threaded file copying and archive unpacking (.zip, .7z, .rar).
- **Core Technology:** Python 3.9+
- **GUI Framework:** `customtkinter` (modern dark-themed UI).
- **CLI Engine:** `argparse` and `tqdm` for progress tracking.
- **Copy Engine:** Multi-threaded `ThreadPoolExecutor` with optimized I/O buffers and optional SHA-256 verification.
- **Unpack Engine:** Utilizes `zipfile`, `py7zr`, and external `7z.exe` (for .rar support).
- **Distribution:** Wrapped as an npm package (`@involvex/supercopy`) and a Windows installer (NSIS).

## Directory Structure
- `supercopy.py`: The main entry point for both GUI and CLI modes.
- `bin/supercopy.js`: Node.js wrapper that launches the compiled `dist/SuperCopy.exe`.
- `build.bat`: Windows batch script to set up a virtual environment and build the executable using PyInstaller.
- `installer.nsi`: NSIS script to generate a Windows installer.
- `requirements.txt`: Python dependencies.
- `package.json`: npm configuration for global installation and release management.
- `dist/`: Contains the compiled `SuperCopy.exe` (generated by `build.bat`).
- `bin/`: Contains the Node.js entry point for the npm package.

## Building and Running

### Development
1. **Prerequisites:** Python 3.9+, Node.js/npm.
2. **Setup & Build:**
   ```shell
   build.bat
   ```
   *This script creates a `.venv`, installs requirements, and generates `dist/SuperCopy.exe`.*

### Running
- **GUI Mode:** Launch `dist/SuperCopy.exe` directly or run `node bin/supercopy.js` (without arguments).
- **CLI Mode:**
  ```shell
  supercopy <source> <destination> [options]
  supercopy <archive> <destination> --unpack
  ```

### Release Process
The project uses `npm run release` to:
1. Increment the version (patch).
2. Run `build.bat`.
3. Generate the NSIS installer (`SuperCopy-Installer.exe`).
4. Publish the package to npm.

## Development Conventions
- **Language:** Python for the core logic, Node.js for the distribution wrapper.
- **Error Handling:** The `CopyEngine` and `UnpackEngine` classes encapsulate operation logic and report errors via callbacks.
- **UI/UX:** Adheres to a dark-themed, modern aesthetic using `customtkinter`.
- **Performance:** Multi-threading is a core feature; the copy engine defaults to `os.cpu_count()` workers.
- **Verification:** Optional SHA-256 integrity checks for copy operations.
